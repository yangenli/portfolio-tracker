# Portfolio Project — performance tracker

Tracks each student's stock portfolio vs. the S&P 500 over the tracking window
(June 22 – Aug 6, 2025) using Yahoo Finance prices.

## Files

```
submissions/            student portfolio files (.xlsx / .csv) — input
compute_portfolios.py   reads submissions, pulls prices, writes docs/results.json
update.bat              double-click to re-run the script
docs/
  index.html            the static web page students open
  results.json          computed results (regenerated by the script)
```


2. Repo → Settings → Pages → Source: `main` branch, `/docs` folder.
3. The page is served at `https://<you>.github.io/<repo>/`.

**Canvas** — upload `docs/index.html` and `docs/results.json` to a Files folder
and link `index.html`, or paste the page into an HTML page/iframe. Keep both
files together so the page can fetch the JSON.

## Methodology / assumptions (all editable at the top of `compute_portfolios.py`)

- Prices are Yahoo Finance **adjusted close** (dividends + splits included = total return).
- Portfolio value = sum(shares × price). **Cumulative return = value_last / value_first − 1.**
  This is scale-invariant, so a portfolio that totals $95k or $119k instead of
  exactly $100k still gets a fair return — only the *weights* matter.
- **Sharpe** = mean(daily excess return) / std × sqrt(252), risk-free = 0% (change `RISK_FREE_ANNUAL`).
- Benchmark = `^GSPC` ("S&P 500"); change `BENCHMARK_TICKER` if you want SPY or `^SP500TR`.
- Ticker typo fixes live in `TICKER_CORRECTIONS` (currently APPL→AAPL, PLNTR→PLTR).
  Every correction is printed when the script runs.

## Data-quality handling

- Tickers Yahoo can't price are dropped and the student is **badged** (⚠) on the page.
- A portfolio where one position exceeds the whole $100k budget (bad share
  count / wrong price) is marked **needs_review** and kept **out of the ranked
  leaderboard** — shown separately so a bad number never sits at #1.
- Students who didn't put an 8-digit ID in their filename are keyed by their name
  slug instead; pin a real ID for them in `STUDENT_ID_OVERRIDES`.
